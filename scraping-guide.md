# Как обходится лимит 2ГИС на ~60 результатов

## Проблема

2ГИС отдаёт максимум ~60 организаций на один поисковый запрос (5 страниц по 12 карточек). Если в городе 3 000+ салонов — стандартная пагинация покрывает лишь малую часть.

## Решение: сеточный краулинг (Geographic Grid)

Основной скрипт — `node_scripts/scrape_grid.cjs`.

### Принцип

Карта города делится на сетку координатных ячеек. Для каждой ячейки выполняется **отдельный** поисковый запрос с высоким зумом. Каждая ячейка — это маленький район, где результатов обычно меньше 60. Объединение ID из всех ячеек даёт полный охват.

```
┌────┬────┬────┬────┐
│ A1 │ A2 │ A3 │ A4 │    Каждая ячейка — отдельный запрос
├────┼────┼────┼────┤    к 2ГИС с координатами центра
│ B1 │ B2 │ B3 │ B4 │    и зумом 14
├────┼────┼────┼────┤
│ C1 │ C2 │ C3 │ C4 │    ID салонов дедуплицируются
└────┴────┴────┴────┘    глобально через Set
```

### Параметры сетки

| Город        | latMin–latMax    | lonMin–lonMax    | Шаг lat | Шаг lon | Примерно ячеек |
|-------------|-----------------|-----------------|---------|---------|---------------|
| Новосибирск | 54.83 – 55.19   | 82.70 – 83.20   | 0.035   | 0.07    | ~80           |
| Барнаул     | 53.30 – 53.42   | 83.60 – 83.85   | 0.035   | 0.07    | ~16           |

### Как работает одна ячейка

1. Формируется URL: `https://2gis.ru/{city}/search/{query}?m={lon},{lat}/{zoom}`
2. Playwright открывает страницу, ждёт загрузки DOM
3. Парсятся все ссылки с `href*="/firm/"` → извлекаются ID
4. Переход на следующую страницу (`/page/2`, `/page/3`...) — до 5 страниц
5. Остановка, если новых ID не найдено или < 10 новых на странице
6. Все ID добавляются в глобальный `Set` для дедупликации

### Результат

Новосибирск: ~80 ячеек × до 60 ID = теоретический охват ~4800. Реально собрано 3400+ уникальных салонов (с учётом дублей между ячейками).

## Альтернативные подходы (в кодовой базе)

### Перехват API маркеров (`scrape_catalog_api.cjs`)

Та же сеточная навигация, но вместо парсинга DOM слушает сетевые ответы 2ГИС через `page.on("response")` на эндпоинт `markers/clustered`. Получает данные (телефоны, рейтинги, адреса) прямо из JSON API без посещения карточек.

### Прямая пагинация API (`scrape_api.cjs`)

Открывает поиск, перехватывает первый ответ `catalog.api.2gis`, извлекает `key` и `region_id`. Затем программно делает `fetch()` с `page=2, 3, ...` и `page_size=12` до исчерпания результатов. Работает в рамках одного viewport — не использует сетку.

### Поисковая пагинация (`scrape_search.cjs`)

Простая пагинация `/search/.../page/N` с фиксированными параметрами карты. Подходит для небольших выборок (барбершопы одного района). Не обходит лимит 60.

## Пайплайн обогащения данных

```
scrape_grid.cjs          ─→  novosibirsk_salons.json (ID + карточки 2ГИС)
scrape_search.cjs        ─→  barnaul_barbershops.json
        │
        ▼
enrich_cards.cjs         ─→  нормализация контактов, обновление рейтингов 2ГИС
        │
        ▼
enrich_yandex.cjs        ─→  поиск на Яндекс.Картах по названию, добавление
        │                     ratingYandex, reviewsYandex, urlYandex
        ▼                     (только для «горячих» — reviews2gis < 50)
merge_hot_leads.cjs      ─→  hot_leads_all.json
                              фильтр: reviews2gis < 50 или null
                              требование: есть telegram или мобильный +79
                              дедупликация по id и url2gis
```

Оркестрация: `pipeline.ps1` — ждёт завершения `enrich_cards`, запускает `enrich_yandex` для каждого файла, затем `merge_hot_leads`.

## Запуск

```bash
# Полный сбор Новосибирска (2-4 часа)
node node_scripts/scrape_grid.cjs --query "салон красоты" --city novosibirsk --out ../novosibirsk_salons.json

# Только сбор ID (быстро, ~30 мин)
node node_scripts/scrape_grid.cjs --ids-only --city novosibirsk

# Обогащение + мёрж
powershell -ExecutionPolicy Bypass -File pipeline.ps1
```
